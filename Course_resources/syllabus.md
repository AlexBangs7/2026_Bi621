### Course description
This is the first of three workshop-style courses in the summer term. In this course, you will learn basic programming skills that you can implement in biological data analysis. This course focuses on Unix shell and Python programming skills, and applies them to basic problem sets relating to parsing large files, e.g. data that is generated by next generation sequencing. Students will also learn how to manage computer resources and work in a shared user environment (Talapas).

### Course goals
- Demonstrate proficiency in navigating a command line environment using Unix/Linux
- Construct a pipeline utilizing basic commands in Unix/Linux
- Write functions in Python which demonstrate basic algorithmic logic
- Understand and interpret common bioinformatics file formats (FASTA, FASTQ, SAM, GFF, GTF, etc.)
- Perform quality assessment, filtering, and initial analysis of high throughput sequencing data
- Utilize various programs or tools to analyze data (including genome assembly)
- Explain how more complex algorithms function (using sequence alignment as an example)
- Successfully submit jobs to a computer cluster and work efficiently and within parameters of a shared user environment
- Demonstrate proficiency at reading, discussing, and critiquing primary research literature

## Course Organziation
### In-class assignments (25% of grade)
During the class, you will be assigned nine "in-class" assignments. It is anticipated that you will be able to complete most of these during class lab time, but some out of class time may be required.

### Problem sets (40% of grade)
Students will be assigned eight problem sets to complete, primarily outside of class time.

### Bioinformatics Python module (5% of grade)
As part of your Python training, you will create and submit your own module containing useful Bioinformatics functions. You should continually update this module as you progress throughout the program.

### Quizzes (10% of grade)
There will be three quizzes during class over material covered in lectures.

### Lab notebook (5% of grade)
Students should keep a lab notebook to be turned in at the end of the course. The goal for this notebook is reproducible science: that is, by reading your lab notebook, you should be able to replicate exactly what you did to obtain your submitted results. It should contain notes/comments on relevant work throughout the course, primarily problem sets PS6, PS7, and PS8 when you are running in silico experiments. This should include detailed information such as software/package version, summarized results from ```/usr/bin/time -v``` from jobs run on Talapas, references to scripts that were run, relevant UNIX commands that were run, etc. You should create a private GitHub repo and invite Leslie (leslie-c) and Jason (jasonsydes) as collaborators.

### Journal club (10% of grade)
Friday afternoons will be journal club-style group discussions on assigned papers. Students are responsible for reading all assigned papers, for writing one “executive summary”, one reflection, and for participating in discussing the paper during the journal club.

### Participation and professionalism (5% of grade)
Students are expected to be on time and engaged during class periods (ask questions, respond to questions, work with classmates, focus on lectures/assignments, and stay for the entire class period). Technology use should be focused on the lecture/assignments.

### Preparing and submitting assignments:
The information required to complete all in-class assignments and problem sets will be given in class. If you find yourself blindly googling how to write a piece of code, you are most likely going about the problem wrong. You must be able to describe how any piece of code you turn in operates. However, you are expected to use available resources (manual pages, fellow students, Stack Overflow, etc.) to troubleshoot/learn when possible.

Students are encouraged to work together and share information. Some students will have a higher skill level than others, and we encourage those students with more experience to help their peers. **However, no direct sharing of code is allowed – each student must write their own code.**

Assignments will be submitted on GitHub in the requested format. Be sure to include your name on the document. Be professional and organized – appropriately name your files and utilize folders. Late assignments will not be accepted.
